   - `createAssessment(userId: string, locale: string, aiDecideMode: boolean): Promise<Assessment>` -- creates new assessment with CREATED status, transitions to IN_PROGRESS
   - `saveAnswer(assessmentId: string, sectionIndex: number, questionKey: string, answer: string): Promise<void>` -- loads assessment, parses answers JSON, adds/updates answer, saves back. Updates currentSection. Sets status to IN_PROGRESS if was CREATED.
   - `saveFollowUpAnswer(assessmentId: string, sectionIndex: number, questionKey: string, answer: string): Promise<void>` -- same pattern but for followUpAnswers JSON field
   - `transitionState(assessmentId: string, newStatus: AssessmentStatus): Promise<void>` -- validates transition using VALID_TRANSITIONS, throws on invalid
   - `getOrCreateAssessment(userId: string, locale: string): Promise<Assessment>` -- finds latest incomplete assessment for user, or returns null (UI handles creation). Used by page.tsx.
   - All actions use `prisma` from `@/lib/db/prisma` and validate auth via session check

   - Server component that checks auth session via better-auth
   - Calls `getOrCreateAssessment()` or loads existing assessment
   - Passes assessment data (or null) to `<AssessmentFlow>` client component
   - Passes locale and user role to client component

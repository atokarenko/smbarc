<task type="auto">
  <name>Task 1: AI prompts + server actions for follow-ups and scoring</name>
  <files>src/lib/assessment/prompts.ts, src/app/[locale]/(app)/scan/actions.ts</files>
  <action>
1. Create `src/lib/assessment/prompts.ts`:
   - `buildFollowUpPrompt(sectionName: string, sectionAnswers: Record<string, string>, previousLevel: string | null, role: string, locale: string): { system: string, prompt: string }`:
     - System prompt establishes AI as business assessment assistant
     - Includes role context (CEO=strategic, COO=operational, CTO=technical)
     - Instructs to generate 1-3 follow-up questions based on answer quality and gaps
     - Includes comprehension level detection: if previousLevel is null (first section), detect from answers; otherwise maintain/adjust
     - Language instruction: respond in Russian if locale="ru", else English
     - Instructs to provide 2-3 suggestion chips per follow-up question, adapted to detected comprehension level
   - `buildScoreAndRoadmapPrompt(allAnswers: Record<string, string>, formulaScores: Record<string, number>, role: string, locale: string): { system: string, prompt: string }`:
     - System prompt: AI maturity assessment analyst
     - Role shapes narrative angle (same data, different framing)
     - Formula scores provided as guidelines (AI may adjust +/-10 based on qualitative analysis)
     - Instructs 3-5 automation roadmap items with priorities
   - `buildRiskAndRoiPrompt(allAnswers: Record<string, string>, formulaScores: Record<string, number>, role: string, locale: string): { system: string, prompt: string }`:
     - System prompt: AI risk and ROI analyst
     - 3-5 risks covering legal, financial, reputational, operational, data categories
     - ROI with conservative ranges, visible assumptions, confidence levels
   - All prompts use .describe() context from Zod schemas for field guidance

2. Add to `src/app/[locale]/(app)/scan/actions.ts` (extend existing file):
   - `generateFollowUpQuestions(assessmentId: string, sectionIndex: number): Promise<FollowUpQuestion[]>`:
     - Load assessment, parse answers for current section
     - Call `generateText` with `Output.object({ schema: followUpSchema })` using prompt from prompts.ts
     - Store generated questions in assessment.followUpQuestions JSON (keyed by section)
     - Store detected comprehensionLevel on assessment record
     - Transition state to AI_FOLLOWUP
     - Return the follow-up questions
   - `generateSuggestionChips(assessmentId: string, sectionIndex: number, questionIndex: number): Promise<string[]>`:
     - Load assessment comprehension level and previous answers
     - Generate 2-3 contextual suggestion chips via a lightweight AI call (or derive from follow-up schema)
     - Return suggestions array
   - `completeAssessment(assessmentId: string): Promise<AssessmentResults>`:
     - Transition state to CALCULATING
     - Load all answers and follow-up answers
     - Calculate formula scores via `calculateDimensionScores`
     - Run BOTH AI calls in parallel with `Promise.all`:
       - Call 1: `generateText` with `Output.object({ schema: scoreAndRoadmapSchema })` using score+roadmap prompt
       - Call 2: `generateText` with `Output.object({ schema: riskAndRoiSchema })` using risk+ROI prompt
     - Merge results into `AssessmentResults` shape matching demo-data.ts interface
     - Store results JSON on assessment record
     - Transition state to COMPLETE
     - Return results
   - Error handling: wrap AI calls in try/catch. On failure, log error, transition state back to IN_PROGRESS (not stuck in CALCULATING), throw user-friendly error.
  </action>

  </verify>
